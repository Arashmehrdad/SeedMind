## Decision boundary

NDNRA must be completed and proven as a standalone research architecture before it is compared with or integrated into the original SeedMind roadmap.

This audit therefore excludes:

- original-roadmap milestone scheduling;
- Nursery production integration;
- production action authority;
- autonomous proposal promotion;
- experiment scheduling or execution;
- any claim that the complete SeedMind plan is finished.

## Remaining standalone NDNRA gaps

### Gap 1: unified standalone acceptance gate

There is no single deterministic runner and report that executes the accepted standalone NDNRA evidence set and produces one inspectable pass or fail result.

The gate must aggregate existing experiments rather than silently strengthen their authority.

### Gap 2: combined restart-equivalence proof

There is no one restart test proving that the state required by the standalone acceptance gate reloads with equivalent:

- graph structure;
- learned effects and provenance;
- dormancy and growth state;
- accepted consequence evidence;
- deterministic bounded outcomes;
- zero action-selection and production authority.

Bounded imagination must remain non-persistent unless a separate persistence boundary is explicitly approved. Its closure can be referenced as static acceptance evidence rather than placed into a stronger checkpoint.

### Gap 3: combined corruption and fallback proof

A standalone completion claim requires explicit evidence that corrupted, stale, incomplete, or incompatible combined state cannot produce partial authoritative recovery.

### Gap 4: final cross-capability authority audit

A single final audit must prove that combining accepted components does not introduce:

- autonomous scheduling;
- autonomous execution;
- route recommendation or selection authority;
- production action influence;
- SQLite cognition;
- hidden promotion paths;
- evidence creation from replayed or imagined activity.

### Gap 5: unresolved expanded-architecture research capabilities

The following remain open or only partially represented and must be explicitly resolved or explicitly descoped before claiming complete expanded NDNRA closure:

- learned context-similarity weights rather than fixed transfer weights;
- semantic abstraction above grounded context signatures;
- coordination of multiple simultaneous needs;
- explicit spreading-activation normalization under competing recruitment;
- locally derived representational saturation rather than experiment-supplied saturation values;
- bounded initial-connection policy for newly created neurons;
- long-horizon interference and adaptability evidence across mixed tasks.

These are not blockers for proving the existing first prototype viable. They are blockers to an unqualified claim that every expanded NDNRA research question is solved.

## Acceptance criteria for the next stage

The stage is accepted only when:

1. All existing standalone experiment gates pass inside one deterministic acceptance run.
2. The consolidated result preserves exact component reports and does not replace them with one opaque score.
3. Save and reload reproduce the same accepted structural and behavioural state.
4. A post-restart rerun produces equivalent deterministic acceptance outcomes.
5. Corrupt, incomplete, stale, or incompatible state fails safely without partial authority-bearing recovery.
6. Replayed and imagined artifacts remain non-factual and non-authoritative.
7. No runtime timer, worker, queue, scheduler, executor, live-integration dependency, or production-action authority is introduced.
8. Focused tests and the complete repository quality gates pass.
