# ADR: NDNRA Developmental Network Implementation and Test Plan

Date: 30 June 2026  
Status: Accepted planning boundary; implementation limited to isolated research batches  
Scope: NDNRA post-v0.1 developmental-network research

## Context

The closed standalone NDNRA v0.1 deliberately proved a bounded architecture with one active need, exact grounded contexts, bounded specialist interaction growth, reversible dormancy, persistence, and zero production action authority.

Subsequent architecture review clarified that several deferred limitations should not be solved by adding a central planner, merging experiences into abstract records, creating a new neuron for every experience, deleting dormant knowledge, or allowing an emergent decision to execute directly.

The corrected theory proposes:

- specialised concurrent regions rather than one uniform mind;
- separate experiences connected through need, context, action, outcome, excitation, and inhibition;
- ordinary recruitment of an existing reusable neuron pool;
- homeostatic control of activation, connectivity, repeated recruitment, and rare structural expansion;
- regional child-to-adult plasticity with bounded local reopening;
- dormancy as hibernation and caller-invoked dream maintenance;
- exact reconstruction of the same developmental network after restart;
- protected prohibitions plus learned responsibility;
- typed action proposals separated from an external protected action gateway.

A staged implementation and falsification plan is required before runtime integration can be reconsidered.

## Decision

Adopt `docs/architecture/NDNRA_Developmental_Network_Implementation_and_Test_Plan_v0.2.md` as the governing post-v0.1 research plan.

The plan authorises documentation and isolated implementation work under `src/seedmind/research/ndnra` only. It does not authorise production integration, autonomous action, physical robotics, proof-store merging, SQLite cognition, or modification of the closed v0.1 evidence boundary.

The implementation order is:

1. freeze v0.1 and define v0.2 contracts;
2. build a reusable sparse recurrent experiential substrate;
3. prove associative need-and-context recall without merging experiences;
4. prove specialised concurrent regions and simultaneous needs;
5. prove regional child-to-adult development and local reopening;
6. prove homeostatic control of connectivity and activation;
7. prove hibernation, caller-invoked dream maintenance, and exact restart identity;
8. prove protected conscience, learned responsibility, and inert typed action proposals;
9. combine the mechanisms in a software-only shadow trial.

A future bounded authority pilot is explicitly excluded and requires a separate ADR after the preceding stages pass.

The first implementation authorisation is limited to Stage 0 and Stage 1: contracts, lifecycle identities, a reusable sparse recurrent neuron pool, distributed experience coalitions, and deterministic bounded settling.

## Required evidence

Every stage must include:

- a matched baseline and at least one mechanism ablation;
- deterministic seeded tests;
- explicit pass and falsification conditions;
- resource and retention measurements;
- exact experience and proposal provenance;
- zero SQLite cognitive operations;
- zero production action-authority violations;
- no mutation of closed v0.1 proof stores;
- full repository quality gates before closure.

## Consequences

- The earlier comparison decision remains valid: the original SeedMind master plan remains the product and MVP spine.
- The 82% expanded developmental architecture marker does not increase merely because this plan exists.
- Semantic concept merging is not a prerequisite for the v0.2 associative-recall experiment.
- Generic per-experience neuron creation is not the default learning mechanism.
- Memory-bearing structures are not permanently pruned.
- Dreaming begins as caller-invoked maintenance and cannot create factual evidence or actions.
- Protected prohibitions cannot be weakened by ordinary learning.
- Production curiosity remains the sole production action authority.
- Physical robotics remains deferred.
- Implementation must stop and return to architecture review when a declared falsification condition persists.
